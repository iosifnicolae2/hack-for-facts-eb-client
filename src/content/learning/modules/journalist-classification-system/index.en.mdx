## Introduction

### Reading the Budget's Language

> **Key Insight:** Every budget line tells you WHO spent, FOR WHAT purpose, and WHAT TYPE of spending. Master this three-dimensional code, and budget documents become transparent.

Budget codes are not random numbers â€” they're a precise language that, once learned, lets you read any budget document in Romania like a native speaker.

## Technical Notes for Implementation

### Components Needed

1. **BudgetCodeDecoder** - Parse and explain budget codes
2. **ClassificationBuilder** - Build queries from dropdowns
3. **RedFlagCodeFinder** - Expandable cards with investigation guides
4. **DecodeExercise** - Interactive practice with answer checking
5. **QuizComponent** - Multiple choice with feedback

### Data Requirements

- Full functional classification hierarchy (static JSON)
- Full economic classification hierarchy (static JSON)
- Example budget lines for exercises
- Entity data for red flag searches (API)

### API Endpoints

- `GET /api/classifications/functional` - Full functional tree
- `GET /api/classifications/economic` - Full economic tree
- `GET /api/classifications/decode?code={code}` - Decode single code
- `GET /api/analytics/red-flags?type=opacity` - High 20.30.30 entities
- `GET /api/analytics/red-flags?type=low-investment` - Low Code 71 execution

### Accessibility

- Code decoder input accepts keyboard paste
- Expandable cards have ARIA expanded states
- Dropdown menus keyboard navigable
- Exercise answers announced by screen readers
