# Report and Act

## Concept Introduction

You've found your entity, collected the data, and identified red flags. Now it's time to turn analysis into action. In this final lesson, you'll create a shareable report and plan concrete next steps.

This is where knowledge becomes power. A well-documented analysis with clear questions is the foundation of effective civic engagement.

> **Key Insight:** Data without action changes nothing. A citizen who asks informed questions can change everything.

### Component Specification: QuestionBuilder

```typescript
/**
 * QuestionBuilder Component
 *
 * PURPOSE: Generate contextual questions based on red flags found,
 * allowing users to customize and save their inquiry list.
 */
interface QuestionBuilderProps {
  readonly redFlagResults: RedFlagResults
  readonly entityName: string
  readonly onQuestionsComplete: (questions: GeneratedQuestions) => void
}

interface QuestionTemplate {
  readonly id: string
  readonly triggerRedFlag: string
  readonly questionText: string
  readonly placeholders: ReadonlyArray<{
    key: string
    description: string
  }>
}

interface GeneratedQuestions {
  readonly suggested: ReadonlyArray<{
    readonly template: string
    readonly filled: string
    readonly isSelected: boolean
  }>
  readonly custom: ReadonlyArray<string>
}

/**
 * STATE MANAGEMENT:
 * - suggestedQuestions: QuestionTemplate[] (filtered by red flags)
 * - selectedQuestions: Set<string>
 * - customQuestions: string[]
 * - isComplete: boolean
 *
 * STATE TRANSITIONS:
 * - Component loads -> filter templates by detected red flags
 * - User toggles question -> add/remove from selected set
 * - User types custom -> add to custom questions array
 * - At least 1 question selected -> enable completion
 *
 * USER INTERACTIONS:
 * - Checkbox to select/deselect suggested questions
 * - Text input for custom questions
 * - "Add another" button for additional custom questions
 * - Drag to reorder questions (optional)
 * - "Copy all" button for clipboard
 *
 * VISUAL STATES:
 * - Suggested question selected: highlighted with checkmark
 * - Custom input active: focus ring, placeholder text
 * - Complete: all questions visible, export options shown
 *
 * ANIMATIONS:
 * - Question selection: checkbox fill + subtle highlight (150ms)
 * - New custom question: slide-down entry (200ms)
 * - Copy confirmation: checkmark flash (300ms)
 *
 * ACCESSIBILITY:
 * - role="group" for question categories
 * - Checkboxes have aria-describedby for question text
 * - Custom inputs have clear labels
 * - Announce question count updates
 */
```

### Component Specification: ReportBuilder

```typescript
/**
 * ReportBuilder Component
 *
 * PURPOSE: Compile all collected data, red flags, and questions
 * into a formatted, shareable report document.
 */
interface ReportBuilderProps {
  readonly entityData: SelectedEntity
  readonly budgetData: CollectedBudgetData
  readonly redFlagResults: RedFlagResults
  readonly questions: GeneratedQuestions
  readonly onReportGenerated: (report: GeneratedReport) => void
}

interface GeneratedReport {
  readonly id: string
  readonly createdAt: Date
  readonly entity: SelectedEntity
  readonly summary: {
    readonly totalBudget: number
    readonly population: number
    readonly perCapita: number
  }
  readonly findings: {
    readonly overallExecution: number
    readonly investmentExecution: number
    readonly personnelShare: number
    readonly topFunctions: ReadonlyArray<{ name: string; percentage: number }>
  }
  readonly redFlags: ReadonlyArray<{
    readonly severity: 'warning' | 'red_flag'
    readonly description: string
  }>
  readonly questions: ReadonlyArray<string>
  readonly actions: ReadonlyArray<{
    readonly type: 'foia' | 'meeting' | 'alert' | 'share'
    readonly description: string
    readonly isChecked: boolean
  }>
}

/**
 * STATE MANAGEMENT:
 * - reportData: GeneratedReport (compiled from props)
 * - editMode: boolean
 * - selectedActions: Set<string>
 * - exportFormat: 'pdf' | 'html' | 'link'
 *
 * STATE TRANSITIONS:
 * - Component mounts -> compile report from all props
 * - User edits field -> update reportData
 * - User checks action -> add to selectedActions
 * - User clicks export -> generate in selected format
 *
 * USER INTERACTIONS:
 * - Click field to edit (inline editing)
 * - Checkbox for each recommended action
 * - Export buttons: PDF, Share Link, Print
 * - "Edit" toggle for review mode vs. edit mode
 *
 * VISUAL STATES:
 * - View mode: clean, print-ready layout
 * - Edit mode: fields have edit indicators
 * - Exporting: loading spinner on button
 * - Exported: success confirmation with copy-to-clipboard
 *
 * ANIMATIONS:
 * - Field edit: border fade-in (150ms)
 * - Export success: checkmark animation (300ms)
 * - Share link copied: tooltip slide-up (200ms)
 *
 * ACCESSIBILITY:
 * - role="document" on report container
 * - Sections have proper heading hierarchy
 * - Export buttons have clear aria-labels
 * - Print styles for accessibility
 */
```

**Data Integration:**
- Report storage: `POST /api/reports` (optional, for persistence)
- Share link generation: Creates unique shareable URL
