# Report and Act

## Concept Introduction

You've found your entity, collected the data, and identified red flags. Now it's time to turn analysis into action. In this final lesson, you'll create a shareable report and plan concrete next steps.

This is where knowledge becomes power. A well-documented analysis with clear questions is the foundation of effective civic engagement.

> **Key Insight:** Data without action changes nothing. A citizen who asks informed questions can change everything.

### Component Specification: QuestionBuilder

```typescript
/**
 * QuestionBuilder Component
 *
 * PURPOSE: Generate contextual questions based on red flags found,
 * allowing users to customize and save their inquiry list.
 */
interface QuestionBuilderProps {
  readonly redFlagResults: RedFlagResults
  readonly entityName: string
  readonly onQuestionsComplete: (questions: GeneratedQuestions) => void
}

interface QuestionTemplate {
  readonly id: string
  readonly triggerRedFlag: string
  readonly questionText: string
  readonly placeholders: ReadonlyArray<{
    key: string
    description: string
  }>
}

interface GeneratedQuestions {
  readonly suggested: ReadonlyArray<{
    readonly template: string
    readonly filled: string
    readonly isSelected: boolean
  }>
  readonly custom: ReadonlyArray<string>
}

/**
 * STATE MANAGEMENT:
 * - suggestedQuestions: QuestionTemplate[] (filtered by red flags)
 * - selectedQuestions: Set<string>
 * - customQuestions: string[]
 * - isComplete: boolean
 *
 * STATE TRANSITIONS:
 * - Component loads -> filter templates by detected red flags
 * - User toggles question -> add/remove from selected set
 * - User types custom -> add to custom questions array
 * - At least 1 question selected -> enable completion
 *
 * USER INTERACTIONS:
 * - Checkbox to select/deselect suggested questions
 * - Text input for custom questions
 * - "Add another" button for additional custom questions
 * - Drag to reorder questions (optional)
 * - "Copy all" button for clipboard
 *
 * VISUAL STATES:
 * - Suggested question selected: highlighted with checkmark
 * - Custom input active: focus ring, placeholder text
 * - Complete: all questions visible, export options shown
 *
 * ANIMATIONS:
 * - Question selection: checkbox fill + subtle highlight (150ms)
 * - New custom question: slide-down entry (200ms)
 * - Copy confirmation: checkmark flash (300ms)
 *
 * ACCESSIBILITY:
 * - role="group" for question categories
 * - Checkboxes have aria-describedby for question text
 * - Custom inputs have clear labels
 * - Announce question count updates
 */
```

```typescript
/**
 * ReportBuilder Component
 *
 * PURPOSE: Compile all collected data, red flags, and questions
 * into a formatted, shareable report document.
 */
interface ReportBuilderProps {
  readonly entityData: SelectedEntity
  readonly budgetData: CollectedBudgetData
  readonly redFlagResults: RedFlagResults
  readonly questions: GeneratedQuestions
  readonly onReportGenerated: (report: GeneratedReport) => void
}

interface GeneratedReport {
  readonly id: string
  readonly createdAt: Date
  readonly entity: SelectedEntity
  readonly summary: {
    readonly totalBudget: number
    readonly population: number
    readonly perCapita: number
  }
  readonly findings: {
    readonly overallExecution: number
    readonly investmentExecution: number
    readonly personnelShare: number
    readonly topFunctions: ReadonlyArray<{ name: string; percentage: number }>
  }
  readonly redFlags: ReadonlyArray<{
    readonly severity: 'warning' | 'red_flag'
    readonly description: string
  }>
  readonly questions: ReadonlyArray<string>
  readonly actions: ReadonlyArray<{
    readonly type: 'foia' | 'meeting' | 'alert' | 'share'
    readonly description: string
    readonly isChecked: boolean
  }>
}

/**
 * STATE MANAGEMENT:
 * - reportData: GeneratedReport (compiled from props)
 * - editMode: boolean
 * - selectedActions: Set<string>
 * - exportFormat: 'pdf' | 'html' | 'link'
 *
 * STATE TRANSITIONS:
 * - Component mounts -> compile report from all props
 * - User edits field -> update reportData
 * - User checks action -> add to selectedActions
 * - User clicks export -> generate in selected format
 *
 * USER INTERACTIONS:
 * - Click field to edit (inline editing)
 * - Checkbox for each recommended action
 * - Export buttons: PDF, Share Link, Print
 * - "Edit" toggle for review mode vs. edit mode
 *
 * VISUAL STATES:
 * - View mode: clean, print-ready layout
 * - Edit mode: fields have edit indicators
 * - Exporting: loading spinner on button
 * - Exported: success confirmation with copy-to-clipboard
 *
 * ANIMATIONS:
 * - Field edit: border fade-in (150ms)
```

## What If Things Don't Go As Planned?

Civic engagement isn't always straightforward. Here's how to handle common obstacles:

<ExpandableHint trigger="Officials don't respond to questions at meetings">

**Don't give up — escalate strategically:**

1. **Document the non-response** — Note the date, meeting, question asked, and who didn't respond. This creates a record.

2. **Follow up in writing** — Send the same question via email or official correspondence. Written requests create accountability trails.

3. **Convert to FOIA** — If verbal questions are ignored, file a formal information request under Law 544/2001. They have 10 days to respond (30 for complex requests).

4. **Involve others** — Share your question with local journalists, civic groups, or other citizens who can amplify the ask.

5. **Attend multiple meetings** — Persistence matters. Ask the same question across several sessions until you get an answer.

**Remember:** Non-response is itself data. Officials who consistently avoid specific questions may have something to hide.

</ExpandableHint>

<ExpandableHint trigger="Your FOIA request is denied or ignored">

**Law 544/2001 gives you enforcement rights. Use them:**

**If your request is ignored (no response within 10 days):**
1. File an administrative complaint with the institution's leadership
2. Send a reminder citing the legal deadline violation
3. If still ignored after 30 days, escalate to the Prefect (for local authorities) or file a court complaint

**If your request is denied:**
1. **Request the denial in writing** — They must explain why under the law
2. **Check if the exemption is valid** — Personal data, national security, and ongoing investigations are legitimate; "we don't want to share" is not
3. **File an administrative appeal** within 30 days of denial
4. **Escalate to court** — You can sue under Law 544/2001 for improper denial (administrative court, no lawyer required for simple cases)

**Common invalid denial reasons:**
- "The data is not public" — Budget execution data IS public by law
- "We don't have it in that format" — They must provide what exists
- "You must explain why you want it" — Law 544 requires NO justification from requesters

**Get help:** Contact the Ombudsman (Avocatul Poporului) if you face persistent obstruction.

</ExpandableHint>

<ExpandableHint trigger="Your analysis seems to reveal serious problems">

**Finding potential wrongdoing? Here's how to proceed responsibly:**

1. **Verify your findings** — Double-check data sources and calculations. Is it really an anomaly, or a data entry error?

2. **Look for context** — Could there be a legitimate explanation? Check for budget rectifications, pandemic impacts, or EU project cycles.

3. **Document everything** — Save screenshots, download reports, note URLs and dates. Evidence can disappear.

4. **Consult before reporting** — Talk to local journalists or civic organizations. They can help assess whether this warrants formal action.

**If you decide to report:**
- **Procurement irregularities** → CNSC (National Council for Solving Complaints)
- **Suspected corruption** → DNA (National Anticorruption Directorate) at dna.ro
- **Financial mismanagement** → Court of Accounts (Curtea de Conturi)
- **Integrity violations** → ANI (National Integrity Agency)

**Protect yourself:**
- You can report anonymously to most institutions
- Whistleblower protection law (571/2004) applies if you're an employee
- Document your own actions in case you need to show good faith

</ExpandableHint>

<ExpandableHint trigger="You're not sure if your findings are significant">

**Uncertainty is normal. Here's how to calibrate:**

**Signs your findings ARE significant:**
- Multiple red flags in the same entity
- Same pattern repeating across years
- Execution rates far outside normal ranges (below 50% or above 150%)
- Large discrepancies between similar entities
- Patterns that match known corruption indicators

**Signs your findings might need more context:**
- Single year anomaly (could be one-off circumstances)
- Borderline values (55% execution, 62% personnel)
- Missing comparison data
- Sector-specific issues you don't fully understand

**What to do when uncertain:**
1. **Compare more** — Add 2-3 more similar entities to your analysis
2. **Check more years** — Is this pattern consistent or a one-time event?
3. **Ask experts** — Journalists, economists, or civic organizations can help interpret
4. **File a FOIA** — Sometimes the best way to understand is to ask the institution directly

**Default stance:** When in doubt, ask questions. Informed inquiry is always appropriate, even when you're not sure of the answer.

</ExpandableHint>
