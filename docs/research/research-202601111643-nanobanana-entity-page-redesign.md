# Deep Research: Redesigning Entity Budget Page with Nano Banana Pro

## Context

I'm building Transparenta.eu, a public budget transparency platform for Romania. I need to redesign the "Entity Page" which displays comprehensive financial data for a public institution (like a city hall, school, hospital).

## Current Data to Display

The entity page needs to clearly explain:

1. **Executie Bugetara (Budget Execution)**
   - Planned budget vs actual spending
   - Budget utilization rate
   - Quarterly/monthly breakdown
   - Year-over-year comparison

2. **Income/Revenue (Venituri)**
   - Revenue sources breakdown (taxes, transfers, own revenue)
   - Spending by functional classification (education, health, administration)
   - Revenue trends over time
   - Comparison to planned revenue

3. **Expenses (Cheltuieli)**
   - Spending by functional classification (education, health, administration)
   - Spending by economic classification (salaries, goods, capital)
   - Expense trends and comparisons

4. **Inter-institutional Transfers**
   - Money received from other institutions
   - Money transferred to other institutions
   - Transfer flow visualization (Sankey-like)

5. **Entity Metadata**
   - Institution name, type, location
   - Contact information
   - Hierarchy (parent institution)

## Target Audience

- Public sector employees reviewing budgets
- Independent journalists investigating public spending
- Citizens wanting transparency on local government finances
- Researchers analyzing budget patterns

## Design Requirements

- Clean, professional dashboard aesthetic
- Data-dense but not overwhelming
- Clear visual hierarchy
- Mobile-responsive considerations
- Accessible (high contrast, not color-only encoding)
- Should explain complex financial data to non-experts
