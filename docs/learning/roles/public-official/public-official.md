## Module 1: The Public Official's Role

**Learning objective:** Understand your position within the public finance system and its ethical foundations.

### Your role in the system

```
┌─────────────────────────────────────────────────────────────┐
│                    PUBLIC TRUST                             │
│                         │                                   │
│     Citizens ──────────►│◄─────── Taxpayers                │
│                         │                                   │
│                         ▼                                   │
│              ┌─────────────────────┐                       │
│              │   PUBLIC OFFICIAL   │                       │
│              │                     │                       │
│              │  • Steward of       │                       │
│              │    public resources │                       │
│              │  • Agent of         │                       │
│              │    transparency     │                       │
│              │  • Guardian of      │                       │
│              │    legality         │                       │
│              └─────────────────────┘                       │
│                         │                                   │
│                         ▼                                   │
│     ┌───────────────────┼───────────────────┐              │
│     │                   │                   │              │
│     ▼                   ▼                   ▼              │
│  Efficiency        Effectiveness      Accountability       │
│  (cost control)    (results)          (transparency)       │
└─────────────────────────────────────────────────────────────┘
```

### Core principles of public finance (Legea 500/2002, Chapter II)

| Principle | Legal basis | Your obligation |
|-----------|-------------|-----------------|
| **Publicitate** | Art. 9 | Proactively disclose budget information |
| **Unitate** | Art. 10 | All revenues and expenses in one budget |
| **Universalitate** | Art. 8 | Gross recording of all transactions |
| **Anualitate** | Art. 11 | Budget approved yearly, respect timeframes |
| **Specializare** | Art. 12 | Spend only for approved purposes |
| **Legalitate** | Art. 14(2) | All actions must have legal basis |

### The hierarchy of budget authority

```
Ordonator principal de credite (OPC)
    │   • Minister, Primar, Președinte CJ
    │   • Full responsibility for budget execution
    │   • Delegates to subordinates
    │
    ├── Ordonator secundar de credite (OSC)
    │       • Intermediary level (e.g., ISJ for schools)
    │       • Coordinates subordinate units
    │       • Reports to OPC
    │
    └── Ordonator terțiar de credite (OTC)
            • Operational units (schools, hospitals, etc.)
            • Executes within approved limits
            • Reports to OSC/OPC
```

### Personal liability framework

| Area | Risk | Mitigation |
|------|------|------------|
| Financial | Personal liability for illegal payments | Verify legal basis before approval |
| Administrative | Disciplinary sanctions | Follow procedures, document decisions |
| Criminal | Abuse of office, embezzlement | Segregation of duties, transparency |
| Political | Loss of position | Professional, non-partisan execution |

> **Try it on Transparenta.eu:**
> Open [Entity Analytics](/entity-analytics) and search for your institution. Review how your budget execution compares to similar entities — this is what oversight bodies see.

**Key Takeaways:**

- You are a steward of public resources, accountable to citizens
- Six core budget principles are legally mandated
- Personal liability extends to financial, administrative, and criminal areas
- The ordonator hierarchy defines your authority and reporting relationships

## Module 2: Legal Framework Mastery

**Learning objective:** Navigate the legal architecture governing public budgets.

### Primary legislation hierarchy

```
Constituția României
    │   Art. 137-139 — Budget principles
    │
    ├── Legi organice (Organic Laws)
    │       │
    │       ├── Legea 500/2002 — Public Finance (as amended by Legea 270/2013)
    │       │       • State budget framework
    │       │       • Budget cycle and deadlines
    │       │       • Execution rules
    │       │
    │       ├── Legea 273/2006 — Local Public Finance
    │       │       • Local budget framework
    │       │       • Own revenues, transfers
    │       │       • Borrowing (max 30% debt service ratio)
    │       │
    │       └── Legea 69/2010 — Fiscal Responsibility
    │               • Deficit limits (3% GDP Maastricht)
    │               • Fiscal Council
    │               • Medium-term framework (3 years)
    │
    ├── Legi anuale (Annual Laws)
    │       ├── Legea bugetului de stat
    │       └── Legea bugetului asigurărilor sociale
    │
    └── Legislație secundară
            ├── HG, OG, OUG — Government decisions/ordinances
            ├── Ordine MF — Ministry of Finance orders
            └── Norme metodologice — Implementation norms
```

### Key legal instruments

| Law/Regulation | Governs | Key provisions |
|----------------|---------|----------------|
| **Legea 500/2002** | State budget | Budget cycle, credit types, execution |
| **Legea 273/2006** | Local budgets | Local autonomy, transfers, borrowing |
| **Legea 98/2016** | Public procurement | Procedures, thresholds, exceptions |
| **Legea 544/2001** | Transparency | FOIA obligations, 10/30 day response |
| **Legea 52/2003** | Public participation | Consultation requirements |
| **Legea 82/1991** | Accounting | Accounting standards, 5-year retention |
| **OG 119/1999** | Internal control | Control environment, CFPP |
| **Legea 672/2002** | Internal audit | Audit function, independence |
| **OMFP 1792/2002** | Commitment control | ALOP phases, CFPP visa |
| **OMFP 2861/2009** | Inventories | Asset management |

> **Important:** Note that OG 119/1999 is a Government Ordinance (OG), not an Emergency Ordinance (OUG). This distinction matters for legal hierarchy.

### Legal hierarchy in practice

When rules conflict, apply in this order:

1. Constitution
2. EU regulations (direct effect)
3. Organic laws
4. Ordinary laws
5. Government ordinances (OG, OUG)
6. Government decisions (HG)
7. Ministerial orders
8. Internal procedures

> **Try it on Transparenta.eu:**
> Use the [Classifications](/classifications/functional/) section to see how legal requirements translate into budget structure. Understanding classification is understanding the law in practice.

**Key Takeaways:**

- Legea 500/2002 (as amended 2013) is your primary reference for state budget procedures
- Legea 273/2006 governs local budgets with a 45-day approval deadline after state budget
- Document retention is now 5 years (not 10) per Legea 36/2023 amendments to Legea 82/1991
- OG 119/1999 (not OUG) governs internal control

## Module 3: Budget Classification System

**Learning objective:** Master the classification system for accurate planning and reporting.

### The three-dimensional classification

| Dimension | Question | Example |
|-----------|----------|---------|
| **Organizational** | WHO spends? | Ministerul Educației → ISJ Sibiu → Liceul X |
| **Functional** | FOR WHAT purpose? | 65.02.04.01 = Secondary education |
| **Economic** | WHAT TYPE of spending? | 71.01.01 = Construction |

### Functional classification (COFOG-adapted per OMFP 1954/2005)

| Code | Category | Common subcategories |
|------|----------|---------------------|
| **51.02** | Autorități publice | Administration, legislative |
| **54.02** | Alte servicii publice | Debt transactions, transfers |
| **55.02** | Dobânzi | Internal and external debt interest |
| **61.02** | Ordine publică | Police, justice, prisons |
| **65.02** | Învățământ | Pre-school through university |
| **66.02** | Sănătate | Hospitals, ambulatory, drugs |
| **67.02** | Cultură, recreere | Libraries, sports, religion |
| **68.02** | Asistență socială | Pensions, social assistance |
| **70.02** | Locuințe, servicii | Housing, water, lighting |
| **74.02** | Mediu | Waste, sewerage, pollution |
| **80.02** | Acțiuni economice | Agriculture, energy, mining |
| **84.02** | Transporturi | Roads, urban transport |

> **Note:** The ".02" suffix indicates local budget classification. State budget uses ".01" suffix.

### Economic classification structure

```
TITLU (Title) — Major category
    └── ARTICOL (Article) — Subcategory
            └── ALINEAT (Line item) — Specific type

Key codes to know:

TITLU 10 — Cheltuieli de personal
    ├── Art. 10.01 — Cheltuieli salariale în bani
    │       ├── 10.01.01 — Salarii de bază
    │       ├── 10.01.05 — Sporuri condiții de muncă
    │       ├── 10.01.06 — Alte sporuri
    │       └── 10.01.30 — Alte drepturi salariale
    └── Art. 10.03 — Contribuții

TITLU 20 — Bunuri și servicii
    ├── Art. 20.01 — Bunuri și servicii
    │       ├── 20.01.01 — Furnituri de birou
    │       ├── 20.01.03 — Încălzit, iluminat
    │       └── 20.01.30 — Alte bunuri și servicii
    ├── Art. 20.02 — Reparații curente
    └── Art. 20.30.30 — Alte cheltuieli (scrutinize carefully!)

TITLU 71 — Active nefinanciare (Capital investments)
    ├── Art. 71.01 — Construcții
    ├── Art. 71.02 — Mașini, echipamente, transport
    └── Art. 71.03 — Mobilier, aparatură

TITLU 81 — Rambursări de credite
    ├── Art. 81.01 — Rambursări credite externe
    └── Art. 81.02 — Rambursări credite interne
```

## Module 4: Strategic Planning and Budget Preparation

**Learning objective:** Develop sound budget proposals aligned with strategic priorities.

### The annual budget calendar (Legea 500/2002 as amended by Legea 270/2013)

```
IULIE
    │   31 iulie: MF sends expenditure limits to Government
    │   
AUGUST
    │   1 august: MF sends framework letter (scrisoare-cadru) to OPCs
    │   Institutions begin preparing proposals
    │
SEPTEMBRIE  
    │   1 septembrie: OPCs submit initial proposals to MF (Art. 34(1))
    │   15 septembrie: Final proposals after negotiations (Art. 34(5))
    │   30 septembrie: MF submits consolidated draft to Government
    │
OCTOMBRIE
    │   Government review and inter-ministerial arbitration
    │   
NOIEMBRIE
    │   15 noiembrie: Government submits to Parliament (Art. 35(4))
    │   Parliamentary committee review begins
    │
DECEMBRIE
    │   Parliamentary debates, amendments
    │   Before 28 decembrie: Parliament votes budget law (Art. 37(1))
    │   Publication in Monitorul Oficial
    │
1 IANUARIE
    │   Budget enters into force
    │   Execution begins
```

> **Critical:** Local budgets must be approved within **45 days** of state budget publication (Legea 273/2006, Art. 39(6)).

### Building a budget proposal

**Step 1: Baseline analysis**

```
Current year execution analysis
    ├── What did we actually spend vs. plan?
    ├── Where were we over/under budget?
    ├── What commitments carry forward?
    └── What structural changes are needed?
```

**Step 2: Strategic alignment**

```
Policy priorities
    ├── Government program commitments
    ├── Sectoral strategies
    ├── EU obligations (PNRR milestones)
    └── Local development plans
        │
        ▼
    Budget allocation decisions
```

**Step 3: Needs assessment**

| Category | Method | Documentation required |
|----------|--------|----------------------|
| Personnel | Organigramă × salary grid | Staffing plan, promotions |
| Goods/services | Historical + inflation | Unit costs, quantities |
| Investments | Project-based costing | Feasibility studies, technical docs |
| Transfers | Legal obligations | Beneficiary calculations |
| Debt service | Amortization schedules | Loan agreements |

**Step 4: Nota de fundamentare**

```
Required elements:
    │
    ├── 1. Bază legală — Laws mandating the expenditure
    │
    ├── 2. Descrierea activității — What the money accomplishes
    │
    ├── 3. Fundamentarea calculelor — Unit costs × quantities
    │
    ├── 4. Comparație cu anul precedent — Changes and justification
    │
    └── 5. Indicatori de performanță — Expected outcomes
```

### Common planning errors

| Error | Consequence | Prevention |
|-------|-------------|------------|
| Over-optimistic revenue | Mid-year cuts, arrears | Conservative assumptions |
| Underfunding mandatory expenses | Rectifications required | Prioritize legal obligations |
| Ignoring carry-over commitments | Budget overruns | Track multi-year commitments |
| Copy-paste from previous year | Misaligned priorities | Zero-based review periodically |
| Missing inflation adjustments | Real value erosion | Apply INS deflators |

> **Try it on Transparenta.eu:**
> Use [Entity Analytics](/entity-analytics) to compare your institution's historical execution rates. Low execution in previous years signals planning problems.

**Key Takeaways:**

- Budget calendar deadlines were updated in 2013 — use current dates
- Framework letter arrives August 1; proposals due September 1-15
- Nota de fundamentare must include legal basis, calculations, and performance indicators
- Plan conservatively — over-optimistic budgets lead to mid-year crises

## Module 5: Budget Approval and Authorization

**Learning objective:** Navigate the approval process and understand authorization mechanisms.

### Approval bodies and timelines

| Budget type | Approval body | Key document | Timeline |
|-------------|---------------|--------------|----------|
| State budget | Parliament | Legea bugetului | By December 28 |
| Local budget | Local council | HCL | Within 45 days of state budget |
| Institutional | Ordonator superior | Buget aprobat | After parent budget approved |
| Rectifications | Same as original | Rectificare | As needed, legally constrained |

### Understanding credit types

```
┌─────────────────────────────────────────────────────────────┐
│                    CREDIT TYPES                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  CREDITE DE ANGAJAMENT (Commitment Authority)               │
│  ├── Maximum you can commit (sign contracts)                │
│  ├── Can exceed single year for multi-year projects         │
│  └── Creates legal obligation for future budgets            │
│                                                             │
│  CREDITE BUGETARE (Budget Appropriation)                    │
│  ├── Maximum you can PAY in current year                    │
│  ├── Annual limit, expires December 31                      │
│  └── Cannot exceed commitment authority                     │
│                                                             │
│  Example: 3-year construction project, total 9M lei         │
│  • Credit de angajament: 9M (approved once at start)        │
│  • Credite bugetare: Year 1: 2M | Year 2: 4M | Year 3: 3M  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Quarterly breakdown (Repartizarea pe trimestre)

- Cannot exceed quarterly allocation without approval
- Unused Q1-Q3 can roll to later quarters (with approval)
- Q4 typically larger (investment execution spike)
- Monthly cash planning within quarters

### Credit transfers (Virări de credite) — Art. 47, Legea 500/2002

| Transfer type | Authority | Limit |
|---------------|-----------|-------|
| Within same article (alineate) | Ordonator | Unlimited within rules |
| Between articles (same chapter) | Ordonator | **10% of chapter** |
| Between chapters | Approval body | **10% of chapter + 5% program** |
| Between functional categories | Parliament/Council | Rectificare required |
| To/from personnel (Titlu 10) | **Prohibited** | Cannot increase or transfer out |

> **Critical timing:** Transfers only permitted starting from **Q3** (trimestrul III) per Art. 47.

### When rectifications are required

- Changes exceeding transfer authority
- New programs not in original budget
- Revenue shortfalls requiring cuts
- Emergency situations (via OUG)

> **Try it on Transparenta.eu:**
> Compare quarterly execution patterns in [Entity Analytics](/entity-analytics). Institutions with >50% of spending in Q4 may have planning or capacity issues.

**Key Takeaways:**

- Credit transfers are limited to **10%** (not 20%) at chapter level
- Transfers only allowed starting Q3 — plan ahead
- Personnel allocations cannot be increased or transferred
- Multi-year projects need commitment authority approved upfront

## Module 6: Commitment Control

**Learning objective:** Master the commitment and payment process (ALOP).

### The four phases of expenditure (OMFP 1792/2002)

```
1. ANGAJAMENT (Commitment)
    │   Legal obligation to pay
    │   • Contract signature
    │   • Employment decision
    │   • Grant award
    │   ✓ Requires: Credit de angajament available
    │   ✓ Requires: CFPP visa
    │
2. LICHIDARE (Verification)
    │   Confirm delivery/service rendered
    │   • Goods received and inspected
    │   • Services performed per contract
    │   • Invoices match terms
    │   ✓ Requires: Supporting documents
    │
3. ORDONANȚARE (Payment order)
    │   Authorize the payment
    │   • Ordonator approval
    │   • Complete documentation verified
    │   ✓ Requires: Credit bugetar available
    │   ✓ Requires: CFPP visa
    │
4. PLATĂ (Payment)
        Execute the transfer
        • Treasury processes order
        • Funds leave account
        ✓ Requires: Cash availability
```

### CFPP — Controlul Financiar Preventiv Propriu

```
CFPP Process (per OG 119/1999):
    │
    Document submitted
    │
    ├── Check 1: Legal basis
    │   └── Is there a law/contract authorizing this?
    │
    ├── Check 2: Budget availability
    │   └── Are credits available at correct classification?
    │
    ├── Check 3: Documentation completeness
    │   └── All required supporting docs present?
    │
    ├── Check 4: Calculation accuracy
    │   └── Math correct? Rates/quantities verified?
    │
    └── Decision:
        ├── VIZĂ CFPP → Proceed to next phase
        └── REFUZ DE VIZĂ → Document reasons, escalate
```

### CFPP visa requirements by operation type

| Operation | Documents required | Key checks |
|-----------|-------------------|------------|
| Contract signature | Contract, procurement docs, budget fișă | Legal basis, availability, thresholds |
| Travel order | Ordin deplasare, justification | Budget line, per diem rates |
| Invoice payment | Invoice, reception, contract | Delivery confirmation, amounts |
| Salary payment | State de plată, pontaje | Calculations, deductions |
| Grant/subsidy | Decision, beneficiary docs | Eligibility, legal basis |

### Commitment tracking throughout the year

```
Best practices:
    │
    ├── Register all commitments immediately upon signing
    │
    ├── Track against credit de angajament limit
    │
    ├── Forecast payment timing by month/quarter
    │
    ├── Flag multi-year obligations for future budgets
    │
    └── Monthly reconciliation with accounting
```

> **Try it on Transparenta.eu:**
> Look at the gap between "Angajamente" and "Plăți" for institutions in [Entity Analytics](/entity-analytics). Large gaps indicate commitment management issues.

**Key Takeaways:**

- ALOP is mandatory: Angajament → Lichidare → Ordonanțare → Plată
- CFPP visa required at commitment and payment authorization stages
- CFPP person must be independent — cannot be pressured on visa decisions
- Track commitments continuously to avoid overcommitment

## Module 7: Cash and Payment Management

**Learning objective:** Manage liquidity and ensure timely payments.

### Treasury system operations

```
┌──────────────────────────────────────────────────────────┐
│                    TREZORERIA STATULUI                   │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  Your institution's accounts:                            │
│                                                          │
│  ┌─────────────────────────────────────────────────────┐│
│  │  Cont de cheltuieli (Expenditure account)           ││
│  │  • Receives monthly cash allocation                 ││
│  │  • Payments executed from here                      ││
│  │  • Daily balance monitoring required                ││
│  └─────────────────────────────────────────────────────┘│
│                                                          │
│  ┌─────────────────────────────────────────────────────┐│
│  │  Cont de venituri proprii (Own revenue account)     ││
│  │  • Fees, services, rents                            ││
│  │  • Different spending rules apply                   ││
│  └─────────────────────────────────────────────────────┘│
│                                                          │
│  ┌─────────────────────────────────────────────────────┐│
│  │  Cont pentru fonduri externe (EU funds account)     ││
│  │  • Separate tracking required                       ││
│  │  • Reimbursement-based flow                         ││
│  └─────────────────────────────────────────────────────┘│
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### Cash planning process

**Monthly:**

```
Solicitare deschidere credite bugetare
    │
    ├── Forecast payments for coming month
    ├── Compare to quarterly allocation
    ├── Submit to treasury/coordinating ordonator
    └── Receive cash allocation
```

**Payment prioritization:**

```
Priority order:
    │
    ├── 1. Salaries and social contributions (legal deadlines)
    ├── 2. Utilities and essential services
    ├── 3. Contractual obligations by due date
    └── 4. Other payments within cash availability
```

### Payment execution

| Payment type | Method | Timeline |
|--------------|--------|----------|
| Salaries | Electronic, batch | Monthly, set dates |
| Suppliers | Bank transfer | Net 30-60 days per contract |
| Utilities | Direct debit or transfer | Per contract terms |
| Social contributions | Electronic declaration | Monthly deadlines |
| Cash advances | Cash withdrawal | Justified, accounted within 3 days |

### Avoiding arrears (arierate)

```
Prevention checklist:
    │
    ├── Don't commit beyond budget availability
    │
    ├── Maintain rolling commitment-payment forecast
    │
    ├── Prioritize payments by contractual due date
    │
    ├── Communicate early if cash problems emerge
    │
    └── Report arrears immediately per regulations
```

### Forexebug integration

- Real-time execution tracking required
- Automatic reporting to Ministry of Finance
- Commitment vs. payment reconciliation
- Mandatory for all public institutions

> **Try it on Transparenta.eu:**
> Institutions with growing arrears show specific patterns in [Entity Analytics](/entity-analytics). Monitor your own patterns and compare to peers.

**Key Takeaways:**

- Treasury accounts have different rules — know which you're using
- Payment prioritization follows legal hierarchy (salaries first)
- Arrears must be reported immediately — hiding them creates legal liability
- Forexebug integration is mandatory for real-time transparency

## Module 8: Accounting and Reporting

**Learning objective:** Maintain accurate accounts and produce required reports.

### Accounting framework

```
Public sector accounting (Legea 82/1991):
    │
    ├── Contabilitate de angajamente (Commitment accounting)
    │       • Records when legal obligation arises
    │       • Regardless of payment timing
    │
    ├── Contabilitate bugetară (Budget accounting)
    │       • Tracks against approved appropriations
    │       • Execution vs. plan comparison
    │
    └── Contabilitate patrimonială (Asset accounting)
            • Assets, liabilities, equity
            • Accrual-based (IPSAS influenced)
```

### Chart of accounts structure

| Class | Contents |
|-------|----------|
| 1 | Capital accounts |
| 2 | Fixed assets |
| 3 | Inventory |
| 4 | Third parties (receivables/payables) |
| 5 | Treasury accounts |
| 6 | Expenses |
| 7 | Revenues |
| 8 | Special accounts |

### Reporting calendar

| Report | Frequency | Deadline | Recipient |
|--------|-----------|----------|-----------|
| Execuție bugetară | Monthly | 15th of following month | Ordonator superior |
| Darea de seamă contabilă | Quarterly | 40 days after quarter | MF, Ordonator superior |
| Bilanț contabil | Annual | May 31 (local) / April 30 (state) | MF, Curtea de Conturi |
| Raport de activitate | Annual | Per internal rules | Council, public |
| Declarații fiscale | Monthly | Per ANAF calendar | ANAF |

### Standard execution report structure

```
Raport de execuție bugetară:
    │
    ├── Prevederi inițiale (Initial appropriation)
    │
    ├── Prevederi definitive (After rectifications)
    │
    ├── Angajamente bugetare (Budget commitments)
    │
    ├── Angajamente legale (Legal commitments — signed contracts)
    │
    ├── Plăți efectuate (Payments made)
    │
```

## Module 17: Performance-Based Budgeting

**Learning objective:** Link budget to results.

### From input to outcome budgeting

```
Traditional (Input-based):
    "We spent 10M lei on education"
        │
        ▼
        No information about results

Performance-based:
    "We spent 10M lei on education, which:
     • Served 5,000 students (10,324 lei/student)
     • Achieved 95% graduation rate
     • Reduced dropout from 8% to 5%"
        │
        ▼
        Links money to outcomes
```

### Results chain

```
INPUTS ──► ACTIVITIES ──► OUTPUTS ──► OUTCOMES ──► IMPACT
│              │              │            │            │
Resources      What we        What we      Short-term   Long-term
used           do             produce      changes      changes
│              │              │            │            │
Budget         Teaching       Students     Higher       Reduced
allocation     hours          graduated    employment   poverty
```

### Performance indicator types

| Type | Example | Purpose |
|------|---------|---------|
| Input | Lei per student | Resource allocation |
| Output | Number of graduates | Production measurement |
| Efficiency | Cost per graduate | Resource utilization |
| Effectiveness | % graduates employed | Goal achievement |
| Outcome | Average earnings increase | Impact measurement |

### SMART indicator criteria

- **S**pecific — Clear what is measured
- **M**easurable — Quantifiable with data
- **A**chievable — Realistic target
- **R**elevant — Connected to objectives
- **T**ime-bound — Defined timeline

### Enhanced budget presentation

```
Program budget format:
    │
    ├── Program description
    │       • What are we trying to achieve?
    │       • Who are the beneficiaries?
    │
    ├── Resources requested
    │       • By economic classification
    │       • Linked to specific activities
    │
    ├── Performance targets
    │       • Output indicators with baselines
    │       • Outcome indicators
    │
    └── Results from previous period
            • Actual vs. target
            • Explanations for variances
```

**Key Takeaways:**

- Performance budgeting connects spending to results
- Use the results chain: Inputs → Activities → Outputs → Outcomes → Impact
- SMART indicators are essential for meaningful measurement
- Report variance explanations, not just numbers

## Module 18: Crisis Management

**Learning objective:** Handle budget emergencies effectively.

### Types of budget crises

| Crisis type | Characteristics | Response tools |
|-------------|-----------------|----------------|
| Revenue shortfall | Tax collection below plan | Expenditure cuts, rectification |
| Cash crisis | Budget exists but cash unavailable | Treasury management, borrowing |
| Expenditure overrun | Spending exceeds authorization | Transfers, rectification, cuts |
| External shock | Disaster, pandemic, war | Emergency budgets, OUG |
| Debt crisis | Cannot service debt | Restructuring, bailout |

### Legal tools for adjustment

```
Escalation ladder:
    │
    ├── Virări de credite (Within ordonator authority)
    │       • 10% of chapter provisions
    │       • Only from Q3 onwards
    │       • Cannot touch personnel upward
    │
    ├── Virări between chapters (Approval required)
    │       • Council/Ministry approval
    │       • 10% chapter + 5% program limit
    │
    ├── Rectificare bugetară (Budget amendment)
    │       • Formal amendment law/decision
    │       • Changes approved budget
    │
    └── Ordonanță de urgență (Emergency ordinance)
            • Government emergency powers
            • Exceptional circumstances only
            • Parliamentary ratification required
```

### Reserve mechanisms

| Reserve | Purpose | Limit |
|---------|---------|-------|
| Fond de rezervă bugetară | General contingency | **5% of budget** (local budgets only) |
| Fond de intervenție | Disasters, emergencies | Per annual budget law |
| Contingency lines | Specific risk coverage | As approved |

### Crisis response checklist

```
When crisis emerges:
    │
    ├── 1. Assess situation
    │       • What is the shortfall/problem?
    │       • What is the cause?
    │       • How long will it last?
    │
    ├── 2. Identify options
    │       • What can be cut/delayed?
    │       • What must be protected (salaries, debt)?
    │       • What authority level is needed?
    │
    ├── 3. Communicate
    │       • Inform leadership immediately
    │       • Inform affected departments
    │       • Inform oversight bodies as required
    │
    ├── 4. Implement
    │       • Execute approved adjustments
    │       • Monitor closely
    │       • Document all decisions
    │
    └── 5. Learn
            • Post-crisis review
            • Update procedures
            • Strengthen resilience
```

**Key Takeaways:**

- Credit transfers limited to 10%, only from Q3 onwards
- 5% reserve fund limit applies to **local budgets only**
- Escalate through proper channels — unauthorized adjustments create liability
- Document all crisis decisions for audit trail
