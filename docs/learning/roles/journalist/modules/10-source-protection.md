## Introduction

### Protecting Those Who Trust You

> **Key Insight:** Sources are people who trust you with their careers, sometimes their safety. Never betray that trust â€” even under legal pressure.

Human sources transform budget data into stories. They provide context, confirmation, and access to documents you can't get otherwise. Protecting them is non-negotiable.

## Technical Notes for Implementation

### Components Needed

1. **SourceTypeAssessment** - Interactive source classification
2. **SecurityProtocolChecker** - Self-assessment with scoring
3. **RiskAssessmentTool** - Source risk calculator
4. **ProtectionPlanBuilder** - Guided plan creation
5. **SecureNotes** - Encrypted note storage (optional)

### Data Requirements

- Source type characteristics (static)
- Security best practices (static)
- Legal framework reference (static)
- Risk calculation algorithms

### Security Considerations

- This module's exercises should not store actual source info
- Any notes should be client-side only
- Recommend external secure tools (Signal, etc.)
- Include disclaimers about legal advice

### Accessibility

- Risk levels have text labels (not just colors)
- Checklists keyboard navigable
- Assessment results clearly announced
- No auto-saving of sensitive information
